Why ?

Main usage is for manual and automatic tests when a 3rd party requires a redirect url.

The goal is to get a decent, quick & readable (and parseable) result in a lightweight webpage.

How to use it ?

Visit any https://test-result.dev/whatever page to get a "whatever" message diplayed. Yes this is dynamic!

No needs to check the URL, the message should be big enough.

Examples

  • https://test-result.dev/pass to get a PASS result
  • https://test-result.dev/fail to get a FAIL result
  • https://test-result.dev/inconclusive to get a INCONCLUSIVE result

If you need to display extra informations, pass them as "msg" querystring, for example https://test-result.dev/pending?msg=wait%205%20minutes
